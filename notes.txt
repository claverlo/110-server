# 2. HTTP Methods (Common Ones Used in Flask)

## GET

➡️ Get data

Example:

```
GET /students
```

Meaning: “Give me the students.”

---

## POST

➡️ Create new data

Example:

```
POST /students
```

Meaning: “Add a new student.”

---

## PUT

➡️ Update data

Example:

```
PUT /students/5
```

Meaning: “Update student #5.”

---

## DELETE

➡️ Remove data

Example:

```
DELETE /students/5
```

Meaning: “Delete student #5.”

# 6. Flask

Flask(tool ) = creates the Flask application (the web server).

Without Flask, your Python program usually just runs in the terminal.

Flask connects your Python program to a web address that can be accessed by a browser or by Thunder Client.

# 11. Workflow

```
Flask → starts server
Client → sends request
request → reads client data
Python → processes data
jsonify → sends response back
```

---

# 12. Simple Workflow (with HOW)

## 1️⃣ Flask starts the server

How?

You run the program:

```
python app.py
```

Flask starts a server at:

```
http://127.0.0.1:5000
```

---

## 2️⃣ Client sends request

How?

By typing a URL or sending a request.

Example:

Browser:

```
http://127.0.0.1:5000/hello
```

or Thunder Client

```
GET /api/products
```

---

## 3️⃣ request reads client data

How?

Using this line:

```
request.get_json()
```

Example:

```
data = request.get_json()
```

This reads the JSON sent by the client.

---

## 4️⃣ Python processes the data

How?

Using normal Python code.

Example:

```
products.append(data)
```

or

```
if product["_id"] == product_id:
```

---

## 5️⃣ jsonify sends response back

How?

Using:

```
return jsonify(data)
```

Flask sends the JSON response to the client.

Example response:

```
{
 "success": true
}
```
